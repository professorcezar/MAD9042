# What is Docker

Docker is an open-source project for automating the deployment of applications as portable, self-sufficient containers that can run on the cloud or on-premises. It can make the development process easier and more efficient and solve some problems we have on deploying the application. The main purpose of Docker is to deploy apps in different environments as required. It surpasses the different hardware boundaries in every environment while running applications.

# What is a Container Image

A container image (or just image) is an isolated, shareable packaged application with all the necessary dependencies and configuration. Images live in repository. Like GitHub, a repository can be public or private.

Containers!

  • A container is a running environment for an image
  • It has a virtual file system and it is bound to a port

# Container vs. Virtual Machine

The Docker container does not include the kernel, as a virtual machine. That makes it lighter and easier to deploy and maintain. Instead it uses the kernel features of the host machine.

# Port binding

To use a container you will need to bind the ports from the container to your host machine.

# docker run

Creates a new container from an image. If the image is not available at the host it will try to download it from a registry.

or, using the detached mode

❯ docker run -d redis

# docker pull

Pull an image or a repository from a registry.

❯ docker pull node

# docker stop

Stop one or more running containers.

❯ docker stop 3c88416a7e05
1

# docker start

Start one or more stopped containers.

❯ docker start 3c88416a7e05
1

# docker rm

Remove one or more containers.

❯ docker start 3c88416a7e05
1

# docker rmi

Remove one or more images.

❯ docker start 3c88416a7e05
1

# docker logs

Fetch the logs of a container.

❯ docker start 3c88416a7e05

# docker exec

Run a command in a running container.
